Visual Key

Sprites

Human (user), Main Agent (Optimus Prime), and Sub-Agent (worker drone). All sprites are programmatic pixel art — no image files.

Sprite Glows

Green glow = actively working
Amber glow = waiting for user input
No glow = idle

Connection Line

Vertical line between Human and Agent indicates communication state.

Active: green dashes + arrowhead (animated)
Waiting: pulsing amber dashes
Idle: faint grey dots

Speech Bubble

A pulsing "?" bubble appears near the agent when it awaits user input. See the "Adding dark mode" session box above for a live example.

Sub-Agent Status Dots

Small dots between the sprite and label indicate each sub-agent's state.

Green = actively working
Amber = idle / waiting
Grey = completed

Status Bar

Bottom bar of each session box. Shows state dot, project label, and session name.

my-project
Refactoring auth module
Green dot + text = active
Amber dot + text = waiting
Grey dot + text = idle

Box Border States

Green glow = active session
Amber pulse = waiting for user (animated)
Red outline = stale 24h+ (no activity)
50% opacity = completed session
